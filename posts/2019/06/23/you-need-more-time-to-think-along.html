你需要更多的思考时间

目前每天都七点半准时起, 无论是否工作日, 几个月下来, 觉得时间多了好多.

最近三个月阅读时间多了很多, 无他, 只因每天要早起坐班车, 下班也坐班车, 每天有大概一个半小时在车上, 只能用kindle打发时间.

每天比其他同事差不多早了一个小时到公司, 这段时间用来做什么呢?

我的答案是: 不要立刻开始工作

当然, 每个人选择不一样, 可能这一个小时没人打扰, 编码效率会高一点.

但是, 为什么不将这个时间空出来, 做一些其他事情, 这些事情, 可以起到类似杠杆作用, 撬动你的工作, 提升效率, 减少无畏的时间浪费, 以便更好地享受生活呢?

当下

我们总是被当下的工作所淹没, 即, 注意力被现在的工作抓走. 例如: 昨天有个bug没修, 今天会有个需求会, 下午要和谁对接口, 晚上要上线等等.

而当一天的工作开始后, 你将马不停蹄, 奔波在各种事务中, 忙累了一天之后, 下班回家洗洗睡, 明天继续

你可能某天觉得好累, 厌倦, 生活过成了模式

假设你有更多的时间思考

所以, 假设你每天有一小时时间, 用来做什么?

思考!

一个人思考!

不管是上班前, 还是下班后, 不管是在公司, 还是回家了, 你都有必要给思考留一些时间, 如果一小时太多, 十五分钟就够了.

思考什么呢? 任何事情, 工作或者生活的问题, 抛出来, 思考, 做一些决策和记录

例如: 工作的风格是不是有需要改进的地方? 昨天与人沟通是不是太过激进了? 今天这个方案有个细节似乎有问题? 今天周五晚上去哪浪? 昨天的bug可能是什么原因? 最近有个手工操作频率很高, 自动化要花多久每次能省多少时间?

注意, 这时候不要开 vim 或者 IDE, 不要开文档等等, 不要在意细节, 思考, 得到结论, 记录, 该做的加入todo list.

例如:

- 将xx操作自动化 => 加入到trello的近期事项中
- 晚上去xx 吃饭   => 直接发微信给老婆
- 方案 XX 有问题  => 记录事项到list, 具体确定时注意
- bug可能原因是:xxx   => 将猜测记录到对应issues
- xxx需要优化  => 到对应issue评论

这些都是很发散的问题, 但是有了安静思考的时间, 可以做出更好的决策, 或者有更好的想法.

每件事情, 都会随着时间推移, 变得更好.

另外, 还可以针对近期工作做下梳理, 以及对今日的todo list做下排期;

然后进入第二步

Learn something

第一步, 花的时间可能10分钟不到, 你可以理清楚很多事情.

那么, 剩下的40-50分钟做什么?

学习!

学什么? 跟项目相关的, 跟工作相关, 跟自己思考问题得到结论相关的内容.

例如, 合作项目对方用的xxx技术, 你从来没有接触过, 昨天接入只是扫了眼文档完成接入, 那么你可以google下这个关键词, 然后开始了解, 甚至深入学习.

例如, 昨天遇到一个报错, 解决了但是没有来得及深究, 这时候可以开始深究

你的经验, 主要来自于你的工作, 以及工作相关的领域. 所以, 需要从这些部分学习, 深入, 得到最大化的成长.

注意, 尽量关注跟工作相关的领域, 那些可能跟你工作相关性很低的领域, 除非0)你在不远的将来会涉及 1) 你想转领域了 2) 你感兴趣或者对你成长有帮助 3) 你对本领域已经很有经验了想尝试下(或许有些激进, 个人观点)

例如, 你是个初级的前端开发, 每日忙于工作, 工作所涉及的技术还没有吃透, 看到机器学习很火想学, 每天花一两个小时在上面, 但是工作中用不到, 你短期内又不可能转, 绝大多数人, 最终是跑了一大堆开源项目的hello world, 走马观花了下文档和各种公司. 那么, 每天的这一两个小时相当于是损耗时间, 作用很小(性价比不高), 即使最终要转, 一样需要重头学习积累.

我的经验是:

跟当前工作相关, 才能事半功倍. 这样或许会有些功利, 但你我以及大部分人都是普通人, 精力有限, 用在刀刃上.

知行合一, 从工作内容出发去拓展, 未来实践的概率高, 对个人成长提升也大.

不说那么多, 40分钟过去, 差不多同事都陆续到了, 你只有几分钟时间, 就要进入战斗模式了

进入战斗模式之前, 建议第三步

每日计划

近两年, 个人一直在使用 trello 作为个人看板

每天在开始工作前, review下今天以及本周需要做的事情, 移动下卡片, 标注下deadline, 备注下注意事项等等

然后, 选择一个, 开始干活.

我们总是容易沉溺于繁杂的事务, 而忘了身在何处.

每天独处的思考时间, 可以让我们脱出局中, 而无论是工作还是生活, 都会逐步改善

每天学习, 然后在工作中实践, 才能感受到成长的力量

每日计划, 效率加成
